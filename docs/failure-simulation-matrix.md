# Failure Simulation Matrix

Every module in this repo ends with a **Failure Simulation** table: a set of realistic ways that module's piece of the stack breaks, how to actually break it, how you'd notice, and how to recover. This page is the index — one place to find any of them without hunting through 18 READMEs.

It's also the raw material for Module 18's GameDay drills: the scripted Chaos Mesh experiments there exist specifically for scenarios this table can't already cover by hand with `kubectl`.

Nothing here is theoretical — every row was written by actually running the "how to break it" command against a live lab cluster while building the corresponding module.

## Using this matrix

- **As a study tool**: pick a row you haven't tried, break it for real, see if you can detect and recover without looking at the "Recover" column first.
- **As a GameDay script**: for a group exercise, have one person apply a scenario from this table without telling the others, and have the rest detect + diagnose + recover using only Module 08/09's observability stack — then write it up with [`postmortem-template.md`](postmortem-template.md).
- **As a coverage check**: if you're extending this repo with a new module, add its Failure Simulation rows here too — this file is meant to always be the complete index, not just Modules 01–18.
